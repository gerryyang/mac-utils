## 优化建议

基于当前实现的功能和状态，有以下几点优化建议：

* 性能优化
  + 懒加载和分页: 当仓库数量变多时，可以实现懒加载或分页功能，避免一次性加载所有数据造成页面卡顿
  + 缓存优化: 在客户端缓存已过滤的结果，避免重复过滤相同条件的数据
  + 图片优化: 如果包含仓库图片，可以实现图片懒加载和压缩

* 用户体验改进
  + 过滤器记忆功能: 记住用户最近使用的过滤条件，下次访问时自动应用
  + 响应式设计增强: 进一步优化移动端显示，确保在小屏幕设备上也有良好的体验
  + 深色模式: 添加深色模式切换功能，减轻夜间使用时的视觉疲劳
  + 热门标签推荐: 基于当前趋势动态生成过滤标签推荐

* 功能扩展
  + 用户自定义过滤器: 允许用户创建和保存自己的自定义过滤条件组合
  + 数据导出功能: 提供导出过滤结果为CSV或JSON的选项
  + 订阅功能: 允许用户订阅特定模型或类别的更新，通过邮件通知
  + 搜索功能强化: 添加高级搜索功能，支持组合条件搜索和正则表达式

* 代码质量提升
  + 代码重构: 将主要的过滤逻辑模块化，提高可维护性
  + 单元测试: 为过滤功能添加单元测试，确保不同过滤条件下的正确性
  + 错误处理改进: 添加更健壮的错误处理和恢复机制
  + 性能监控: 添加前端性能监控，收集用户实际使用中的性能数据

* 数据质量改进
  + 增强模型关联识别: 改进对模型相关仓库的识别算法，减少误判
  + 关联数据丰富: 增加仓库与研究论文之间的关联，展示实现了特定论文的仓库
  + 社区热度指标: 添加更多社区互动指标，如Issue活跃度、贡献者数量等
  + 更新频率优化: 根据仓库活跃度动态调整抓取频率

通过以上优化，LLM News 平台将能提供更高效、更个性化的用户体验，同时更好地满足 AI 研究人员和开发者快速获取和筛选相关信息的需求。


# 思路：微信小游戏

需求调研：

请分析下现在比较火的微信小游戏或者抖音小游戏类型有哪些？

微信小游戏用户超10亿，月活5亿，抖音小游戏用户约5亿，角色卡牌和放置类游戏在Q1占主导，策略类在Q2、Q3发力。说明这些类型比较热门。

1. 休闲类如《躺平发育》表现好，还有策略类和社交互动的元素。《躺平发育》是主要投放内容，还有中重度游戏开始崛起，比如模拟经营、脑洞文字等，同时社交互动被重视。玩法与APP的重合度达40%，可能说明一些复杂玩法的小游戏受欢迎。
2. 微信小游戏的社交化和下沉市场，卡牌类游戏《幽游白书：宿命觉醒》进入畅销榜，还有SLG、轻度MMO和塔防等融合元素的新品类。微信的女性用户ARPU值增长，可能说明某些类型在女性中更受欢迎。
3. 抖音的买量指南，提到经典休闲类如三消、模拟经营消耗占比提升，传统重度类如RPG、SLG、放置卡牌占比65%，说明这些类型在买量上投入大，可能比较火。抖音大会数据提到IAP小游戏增长，RPG、SLG等消耗最高，结合直播和达人内容。

微信方面，卡牌、放置、社交休闲类比较火，比如《幽游白书》；抖音方面，休闲类如《躺平发育》是主力，但中重度如RPG、策略类在增长，还有模拟经营、脑洞文字。同时，两者都强调社交化和融合玩法。可能还要提到用户结构，比如抖音大龄男性和年轻女性用户多，影响游戏类型设计。


> 抖音小游戏热门类型

1. 休闲轻量类游戏
  + 代表作品：《躺平发育》《吸油记》《一波超人》
  + 特点：操作简单、单局时间短，适合碎片化娱乐。例如，消除类、合成类（如“二合三消”）、塔防等玩法持续占据主流，尤其在广告投放中占比显著。2024年Q1至2025年Q1，经典休闲类消耗占比提升至7%以上。
  + 用户偏好：年轻女性和大龄男性用户占比较高，游戏动机以放松和快速娱乐为主。

2. 中重度策略与RPG类游戏
  + 代表作品：《别惹农夫》《人生模拟器》
  + 特点：融合策略、放置、模拟经营等元素，注重长线运营和社交互动。例如，放置卡牌、SLG（策略类）和RPG（如仙侠/传奇题材）在2025年Q1买量消耗占比合计接近60%。
  + 趋势：开发方向转向玩法深度化，如加入局外养成、职业搭配等策略性设计，并通过社交机制提升用户留存。

3. 融合创新类游戏
  + 代表玩法：塔防+Roguelike、割草式清怪体验
  + 特点：结合多种玩法机制（如随机性、数值策略），吸引偏好新鲜感的用户。尽管融合类消耗占比有所下降，但头部产品仍通过创新保持竞争力。

4. 社交互动驱动型游戏
  + 趋势：通过直播、短视频达人合作提升曝光，例如抖音小游戏直播达人的月收入可达5万以上，用户复访率依赖侧边栏入口（占67%）。

> 微信小游戏热门类型

1. 卡牌与放置类游戏
  + 代表作品：《幽游白书：宿命觉醒》
  + 特点：传统卡牌玩法融入SLG或轻度MMO元素，强调策略性和收集乐趣。2024年Q1卡牌类占据主导地位，2025年融合放置元素的卡牌游戏（如《春秋玄奇》）进一步丰富市场。

2. 社交休闲类游戏
  + 特点：依托微信社交链，设计排名分享、合作对抗等机制。例如，消除类游戏通过好友助力或PK模式提升用户粘性，女性用户广告ARPU值同比增长70%。

3. 下沉市场驱动的品类
  + 用户群体：中年用户和女性用户占比提升，推动模拟经营、益智解谜等轻度游戏增长。例如，消除类在2024年下半年崛起为市场TOP1品类。

4. 跨端融合类游戏
  + 趋势：微信探索PC端小游戏，模拟RPG等类型在PC端的人均时长和付费渗透率显著提升，形成“多端联动”生态。


> 两大平台的共性趋势

1. 社交化与长线运营

抖音和微信均通过社交机制（如直播、短视频互动、好友排名）延长游戏生命周期。抖音侧边栏入口和微信视频号直播成为关键复访渠道。

2. 重度化与玩法融合

传统手游的玩法（如RPG、SLG）逐渐渗透小游戏领域，开发者通过简化养成线、加速经济循环适配小游戏场景。

3. 广告与内购混合变现

抖音IAP（应用内购）小游戏占比提升，微信则通过IAA（应用内广告）实现30%的变现增长。小额支付和低门槛付费策略成为主流。

> 未来潜力方向

1. AI与UGC内容结合：例如利用AI生成个性化角色，增强玩家参与感。

2. 直播与达人生态：抖音小游戏直播流水已超过达人视频收入，未来或成为核心增长点。

3. 政策红利：平台对首发游戏提供高额补贴（如抖音最高500万配资），激励开发者创新。

**总结：当前微信和抖音小游戏的热门类型均围绕用户碎片化需求展开，但抖音更偏重休闲与中重度融合，微信则依赖社交链和下沉市场。开发者需结合平台特性，平衡玩法创新与用户留存策略。**


## 躺平发育

请介绍下抖音小游戏躺平发育的游戏实现方案，请列出详细的步骤，适合使用 cursor 进行开发的提示词，便于实现一个类似的游戏。

---

以下是基于抖音小游戏《躺平发育》的游戏实现方案及开发步骤，结合其核心玩法和搜索结果中的技术要点，整理为适合使用Cursor等AI辅助工具开发的提示词和实现框架：

一、核心玩法拆解

1. 基础规则

* 双角色对抗：玩家随机分配为“猎梦者”（攻击方）或“睡梦者”（防御方），猎梦者需击破睡梦者的防御门，睡梦者需通过建造塔防和升级装备存活。
* 经济系统：睡梦者通过“睡眠”自动产出金币，金币用于升级床铺（提升金币产出效率）、建造炮塔/维修台等防御设施。
* 塔防机制：炮塔自动攻击猎梦者，维修台修复门耐久值，游戏机提供额外增益。

2. 核心循环

* 睡梦者：睡眠→金币积累→升级装备→抵御攻击→存活至倒计时结束。
* 猎梦者：移动→攻击门→升级属性（攻速/攻击力）→击溃所有睡梦者。

二、技术实现方案

步骤1：环境搭建

* 引擎选择：推荐使用轻量级框架如Phaser.js（H5）或Cocos Creator（跨平台），适配抖音小游戏容器。
* 开发工具提示词："使用Cocos Creator创建一个2D塔防游戏项目，适配移动端竖屏布局，集成抖音小游戏SDK"

步骤2：核心功能开发

1. 角色系统

* 睡梦者：实现角色移动（点击房间选择）、上床动画、金币自动生成逻辑。提示词："用TypeScript实现角色上床后每秒生成金币的逻辑，金币数量随床铺等级提升"
* 猎梦者：AI路径规划（自动寻路至最近门）、攻击动画、属性成长系统。提示词："基于A*算法实现猎梦者自动寻路，攻击门时触发耐久值减少和维修台修复逻辑"

2. 经济与建造系统

* 金币管理：通过计时器动态更新金币数量，关联UI显示。提示词："使用Cocos的schedule实现每2秒增加金币，升级床铺后提升增量参数"
* 塔防建造：点击空地弹出建造菜单，消耗金币生成炮塔/维修台。提示词："实现点击地图格子后弹出建造面板，选择炮塔类型后扣除金币并生成预制体"

3. 战斗与碰撞检测

* 炮塔攻击：基于圆形碰撞检测范围，触发子弹发射和伤害计算。提示词："用Phaser.js的物理引擎检测炮塔与猎梦者的碰撞，触发子弹动画和HP减少"
* 门耐久系统：显示耐久进度条，归零后游戏失败。提示词："使用Tween动画实现门耐久值减少时的进度条动态效果"

步骤3：联机与匹配系统 (可选，可以先实现一个单机的版本)

* 网络架构：采用帧同步或状态同步（推荐WebSocket + Redis）。提示词："使用Socket.io实现多玩家房间匹配，分配猎梦者/睡梦者角色并同步状态"
* 数据同步：同步角色位置、金币数量、建筑状态等关键数据。

步骤4：美术与音效

* 像素风格资源：使用Aseprite制作角色和建筑像素动画，导出为精灵图集。提示词："导出Aseprite动画为JSON+PNG序列帧，在Cocos中加载为SpriteAnimation"
* 音效设计：添加点击按钮、炮塔攻击、门破损等音效，适配移动端压缩格式（MP3/OGG）。

步骤5：性能优化

* 对象池管理：复用炮塔子弹、特效等高频生成对象。提示词："实现对象池管理子弹预制体，避免频繁实例化导致内存泄漏"
* 渲染优化：合并静态建筑节点的Draw Call，使用离屏Canvas处理UI元素。


三、适配抖音小游戏平台

1. SDK集成：接入抖音登录、分享、广告接口（如激励视频复活功能）。提示词："调用抖音小游戏API实现分享截图功能，用户分享后奖励金币"
2. 包体压缩：使用微信SKia图形库或TinyPNG压缩资源，控制包体在20MB以内

四、测试与发布

1. 自动化测试：使用Jest或Cocos Test Framework覆盖核心逻辑。
2. 云真机调试：通过Testin云测平台验证多机型兼容性。
3. 数据监控：接入友盟或Firebase分析用户留存和付费行为。

## 速度与激情的竞速对抗

`问题：`

* 竞技对抗类游戏，主要以 MOBA 和 FPS 类型为热点。赛车竞速类游戏如何实现竞技对抗？
* 不只是「速度」，而是「对抗」? 「竞速」的本质其实就是「对抗」，只是「对抗」的目标从「击杀」转为了「速度」
* 每款竞速游戏里面都有几十上百辆赛车，可真正被玩家玩起来的就这么几辆，造成大量的游戏资源被浪费？引入赛车技能？
* 竞速玩法比较偏单人的体验，如何提高社交，而不是只有一个人爽？


`切入点：`

* **画面风格 -> 更激情的视觉效果**
  + 让玩家在游戏世界，通过画面和音乐可以感受到，**现实世界无法满足的速度与激情**。例如：Q飞手游除了是一款优秀的赛车游戏，带给玩家更多回忆和共鸣的是经久不衰的背景音乐。

* **赛车属性 -> 合作 + 竞争 + 平衡性**
  + 竞速比赛，除了**快起步，氮气足，高时速**的特性外。在 MOBA 竞速场景下，可设计不同技能的赛车或道具，**通过引入赛车技能，使得玩家可以分工协作，组队意愿更强，游戏社交能力自然提升**，另外也可以培养玩家收集更多的车辆，进而提升玩家的养成性，**解决大量的游戏资源被浪费的问题**。例如：在首都高赛车游戏中，在 PK 环节并不是跑的最快的就一定获胜；在马里奥赛车游戏中，落后的玩家可以通过更高概率获取更好的道具来提升自己的名次。

* **玩法多样性 -> 竞速不只是单打独斗，也不只是追求最快的速度**，设计不同的纬度，引入更多可能性。
  + **赛车手 + 领航员**。在现实中的 F1 赛车场景，每个车队的赛车差异不大，决定车队胜负的更多是看车队的配合。通过引入更多的玩家角色和配合方法，让每个角色都能获得自己的成就感。
  + **赛车改装**。通过改装赛车，可以引入不同的玩法，而不只是提升赛车性能。例如：不同的赛车适合不同的赛道，不同的地形，细分不同的场景，引入不同的玩法，使得每辆赛车都具备自己的用武之地。
  + **变形金钢与赛车**。赛车和变形金钢是大多数人童年的回忆，机甲在 Q 飞手游中也被很多人喜爱，可以基于机甲在车与人的形态转换扩展更多的玩法。
  + **汽车杂技**。赛车除了比时速，还可以比技能，类似在《速度与激情》里面主角每次开车起步会将车头抬起；在《头文字D》里的排水渠过弯。在极品飞车集结中，有一个高级技能是钟摆漂移，除了在单局比赛中可以使用此技能，也可以考虑设计专门的技能赛道，让玩家比拼操作技能而不只是速度。例如：侧边行驶，飞跃大峡谷，高速过弯等，可以从其他纬度给玩家带来不同的体验。
  + **血条机制**。很早之前就是天天飞车的玩家，在这款竖屏游戏中，玩家通过借助加速道具来“增加”自己的血条使得行驶更远的距离。这种 PVE 的玩法虽然简单，但是满足了玩家追求“攀比”的目标。可以考虑在现有的玩法中引入类似的设计，可以作为一种数值玩法，类似极品飞车集结里的车库，钓鱼等数值养成玩法。

* **商业化**。好玩的游戏不一定能赚钱。在写实赛车领域，投入大但回报又没有卡牌游戏直接。在满足玩家游戏可玩性的前提下，还需要考虑游戏的付费深度，以及长期运营能力，让玩家持续有爽感，留着更多的长线用户。





## 三角洲特种部队玩法

考虑点：

* 一款基于“搜打撤”的 MOBA 赛车游戏
  + 战术驾驶：适合玩家探索的复杂地形，设计不同的地形，适合不同的车辆
  + 各自搜索资源，并在规定的时间内完成相应的任务




参考《三角洲特种部队》系列的战术性、小队协作和目标导向玩法，结合“搜打撤”（Search-Destroy-Exfiltrate）的核心循环，我们可以设计一款融合了战术驾驶、激烈战斗和高风险撤离的赛车/载具动作游戏：

游戏名：《破晓突围：暗涌行动》 或 《60秒战术竞速》

**核心理念**：将高速载具驾驶与精准的战术打击、高风险撤离相结合。玩家扮演精英“车手佣兵”，驾驶高度改装的战术载具，在危机四伏的城市或特殊区域执行高风险、快节奏的“搜打撤”任务。强调速度、策略、时机和载具改装。

> 搜

* 目标导向：每个任务都有明确的初始目标（如：定位高价值目标车辆、找到被劫持的数据核心、侦察敌方设施弱点）。
* 战术驾驶：驾驶改装车辆快速穿越复杂地形（城市街道、工业区、沙漠公路、雨林哨站等）。利用速度甩开巡逻队，利用地形（窄巷、地下通道、飞跃点）规避或迷惑敌人。
* 环境互动：使用车载扫描仪（消耗能量）侦测隐藏目标、敌方单位位置、可破坏结构或捷径。可能需要短暂停车或低速行驶进行深度扫描（增加风险）。
* 情报收集：可能需要“骇入”特定节点（靠近并维持连接一段时间）获取更精确的目标位置或撤离点变更信息。
* 潜行选项（可选）：部分车辆可装备“光学迷彩”或“声学静音”系统（消耗大、时间短），用于高风险区域的短暂潜入侦察。

> 打

* 武装载具： 玩家的车辆就是移动武器平台。武器种类丰富：
  + 主武器： 车载机枪（弹药无限，过热）、榴弹发射器（抛物线，范围伤害）、电磁脉冲炮（瘫痪敌方载具电子设备）、穿甲机炮（对重甲有效）、小型导弹（锁定，高伤害）。
  + 副武器/装备： 智能地雷/钉刺带部署器、烟雾弹/闪光弹发射器、主动防御系统（拦截导弹）、紧急维修无人机（有限次数）、干扰波（影响锁定）。

* 动态战斗：
  + 高速接战： 与敌方武装车辆、无人机、固定炮台甚至轻型飞行器进行高速追逐战和狗斗。射击需要预判和精准控制。
  + 环境利用： 撞毁敌方路障、引爆炸药桶制造混乱、利用地形制造车祸或卡住敌人。
  + 目标打击： 摧毁特定目标（车辆、设备、建筑支撑结构）。可能需要特定武器或攻击特定弱点。
  + 动态难度： 成功打击会提升“警戒等级”，引来更多、更强的敌人和更严密的封锁。

* 载具状态管理： 车辆会受损（影响速度、操控、武器效能），需要时刻注意生命值和部件状态（轮胎、引擎、武器）。严重受损可能导致任务失败。

> 撤

* 高风险高潮： “打”阶段完成核心目标后，警报拉满，进入最紧张刺激的撤离环节。
* 逃离封锁： 敌人会设置路障、部署重武器、派出精英追击单位（如高速武装直升机、重型装甲车）。地图特定出口或撤离点会被封锁，需要玩家寻找备用路线或强行突破。
* 限时压力： 通常有严格的时间限制到达撤离点（如直升机降落点、地下通道入口、边界线）。
* 最终冲刺： 将驾驶技术发挥到极致，利用所有剩余武器和装备（烟雾掩护、EMP瘫痪追兵、部署地雷拖延）杀出一条血路。
* 撤离点防御： 到达撤离点后，可能需要坚守一小段时间（如等待直升机降落或闸门开启），抵御蜂拥而至的敌人。
* 成功/失败判定： 在时限内成功抵达并守住撤离点即任务成功。车辆被毁、玩家死亡或超时即失败。





## 糖豆人玩法


* 网易的《王牌竞速》就是一个将 MOBA 元素融入赛车游戏的案例。制作人提到他们给赛车加入了"技能"概念，这解决了传统竞速游戏同质化问题，并增强了社交和收集元素。
* 《糖豆人》的关卡设计思路值得借鉴，特别是"水涨爬高"这个关卡的设计过程。开发者谈到了如何平衡混乱和技巧的比例，如何让玩家在犯错后仍有翻盘机会，这些理念可以应用在动态赛道的设计中。
* MOBA 游戏不能固定思路，需要根据每局情况调整策略，这启发我们在赛车游戏中也要设计动态变化的赛道和机制。

设计一个名为"极速领域"的 MOBA 赛车游戏，将传统 MOBA 的三路推塔改为多路径赛道，将英雄定位转化为赛车类型，将技能系统转化为赛道互动能力。参考《糖豆人》的随机关卡机制，每局赛道布局会动态变化，增加变量和可玩性。

基于 MOBA 游戏设计经验与《糖豆人》的动态关卡灵感，设计一款名为《极速领域》（Velocity Arena）的 MOBA 赛车游戏。该游戏将传统 MOBA 的团队策略、角色技能与竞速玩法深度融合，结合动态赛道和战斗机制，打造高变量、强协作的竞技体验。以下是核心设计框架：

* 竞速目标 MOBA 化
  + 赛道设计为多路径分支结构（类似 MOBA 三路推塔），终点改为“核心摧毁”目标。
  + 队伍需分工协作：突击位负责快速突进破坏敌方防御塔（赛道障碍），支援位修复己方赛道设施，坦克位阻挡敌方载具或控制关键区域。

* 角色技能与载具绑定
  + 每位英雄拥有专属赛车及3个技能：
    - 战术技能（短CD）：如放置临时路障、发射钩锁改变对手路线
    - 团队技能（中CD）：如群体加速、修复赛道破损
    - 终极技能（充能释放）：如召唤陨石砸毁赛道区域，或开启全队隐身突袭
  + 参考《王牌竞速》的“技能对抗”思路，技能设计需符合载具特性（越野车擅长地形改造，超跑具备瞬间加速）


* 动态赛道系统：单局变量的核心来源

  + 环境事件机制
    - 赛道每30秒触发全局事件，参考《糖豆人》的随机障碍
  + 可破坏地形与互动机关
    - 赛道中的桥梁、护盾发生器可被技能破坏，改变通行逻辑
    - 团队可占领“能量节点”激活捷径或陷阱（如喷火机关），需分配人员驻守

* 战斗与血量机制：MOBA 式对抗的融入

  + 双重血量系统
    - 载具耐久值：受攻击减少，归零后进入15秒修复状态（期间无法移动）
    - 驾驶员生命值：角色脱离载具后（如占领据点时）暴露，可被敌方技能击杀，复活需30秒
    - 平衡思路：载具技能侧重赛道交互（地形改造/加速），角色技能侧重战斗（射击/治疗）


  + 资源争夺设计
    - 地图中散布“能量晶体”，用于：充能终极技能 / 修复己方赛道 / 临时强化载具（如免疫控制5秒）
    - 逼迫队伍在“速攻”与“资源控制”间抉择

* 团队协作机制：策略大于操作

  + 角色定位与分工
  + 胜负判定多维化
    - 竞速胜利：首个摧毁敌方核心
    - 战术胜利：15分钟内对方载具总耐久归零
    - 结合《糖豆人》的“生存淘汰”机制，圈数落后者将逐步被限制复活次数


> 总结

《极速领域》通过 “动态赛道+技能对抗+团队目标” 三重设计，解决了传统赛车游戏重复度高、MOBA 游戏门槛高的问题。其变量系统（如事件、可破坏地形）深度借鉴《糖豆人》的随机乐趣，而角色分工与资源争夺则延续 MOBA 的策略内核，兼具休闲与硬核竞技潜力。

可行性验证：类似思路已在《王牌竞速》（技能对抗）和《碧蓝幻想VS》（糖豆人式小游戏）得到部分验证36，本方案将其系统化融合，具备较高开发价值。

## 马里奥赛车玩法

上述设计的 Moba 赛车，完全模拟 Moba 的玩法可能不太合理，应该结合赛车本身的玩法和优势设计 Moba 玩法，不能生搬硬套 Moba 的玩法，这样会失去赛车本身的玩法意义。游戏玩法需要满足玩家对竞速的基本玩法，在此基础上添加更多有趣的玩法。可以参考 switch 上的马里奥赛车的一些玩法。

---

重新优化了 MOBA 赛车游戏的设计框架，核心思路是以竞速体验为根基，MOBA 元素为战术变量，避免生硬嫁接。结合《马里奥赛车》的趣味对抗逻辑（道具博弈、尾流、地形互动）和 MOBA 的策略协作，以下是迭代后的设计方案：

* 竞速核心强化：速度与操作的不可替代性
  + 基础驾驶体验优先
    - 保留漂移加速、氮气管理、路线选择等核心竞速操作，参考《马里奥赛车》的“尾流加速”机制：紧跟前车产生气旋后可触发爆发性提速，但需精确控制车距避免碰撞
    - 引入“地形适配”机制：不同载具（越野车/超跑）在特定地形（沙地/公路）有速度修正，鼓励玩家根据赛道特点选车

* 动态赛道设计（糖豆人式变量+马里奥赛道互动）
  + 每局随机触发3-5种赛道事件（如重力反转、黏液蔓延），但事件以环境干扰而非战斗为主，例如：
    - 电磁风暴”：禁用部分攻击技能10秒，迫使玩家专注驾驶
    - “捷径解锁”：队伍占领能量节点后激活隐藏近道，需工程师载具搭建临时桥梁

  + 可破坏地形仅限策略性障碍（如护盾发生器），破坏后永久改变赛道路径，非战斗载具无法直接摧毁


* MOBA 元素轻量化：技能服务于竞速节奏

  + MOBA 元素轻量化：技能服务于竞速节奏
  + 角色定位分工（参考 F1 车队战术）
    - 领航员（突击位）：技能以自提速为主（如“超频引擎”短暂突破极速），承担破风职责
    - 工程师（支援位）：修复赛道、部署临时加速带
    - 战术师（控场位）：释放烟雾干扰视线或部署侦察无人机揭露敌方道具信息

* 资源与目标设计：竞速框架下的 MOBA 式决策

  + 双资源系统驱动策略
    - 氮气：基础加速资源，漂移/腾空可恢复
    - 能量晶体：地图固定点刷新
    - 设计目的：迫使队伍在“抢速”与“资源控制”间权衡，类似 MOBA 打野与推线的平衡

  + 胜负判定多维化
    - 主目标：摧毁敌方终点线前的“能源核心”（需多次冲线攻击）
    - 副目标：单圈领先奖励：每圈首位冲线队伍对核心造成额外伤害 / 资源控制胜利：若15分钟内敌方晶体储备归零，直接获胜

  + 参考《糖豆人》淘汰机制：圈数落后玩家逐渐减少复活次数，最终关停载具


> 总结：竞速为骨，MOBA为翼

这一版设计通过 “三轻一重”原则（轻战斗、轻控制、重策略、重操作）：

* 竞速根基：继承《马里奥赛车》的尾流、道具博弈与地形互动，确保操作深度；
* MOBA 变量：以资源控制、角色分工、动态赛道构建策略维度，参考《荒野竞速》的立体战术设计；
* 平衡保障：技能仅干扰竞速节奏（如降速30%而非定身），终极技能需通过驾驶技术充能，杜绝“无脑砸技能”的 MOBA 化倾向。

最终实现“操作决定下限，策略决定上限”的硬核竞速体验，兼具单局变量与电竞深度。
